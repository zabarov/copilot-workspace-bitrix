# Каталог `results/`

Минимальное хранилище артефактов, которое использует Copilot вместе с агентами.

- `checks/` — сюда Builder складывает вывод проверок (PHPUnit, PHPStan, PHP-CS-Fixer, Deptrac и т.д.). Каждый запуск сохраняется отдельным файлом, чтобы всегда можно было посмотреть логи падений или прогонов.
- `batches/` — заметки по выполненным батчам: `results/batches/<id>.md` описывает, что делал Builder, какие файлы менялись и как прошли проверки. Planner и Documentation читают эти файлы, чтобы продолжить работу.

При необходимости можно расширить структуру (например, добавить `state/` или `snapshot/`), но для простого сценария двух папок достаточно: один каталог хранит отчёты проверок, второй — историю батчей.
