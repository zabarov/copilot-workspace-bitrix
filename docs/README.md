# docs/

Каталог `docs/` хранит постоянные артефакты проекта: архитектуру, ADR, cookbook, OPS и другие документы, которые переживают релизы. Актуализацией занимается агент `50-docs-update` (см. раздел «Роли агентов» в `README.md`), синхронизируя код и батчи с документацией.

Поддерживайте README актуальным: если добавляете новую подпапку или документ, кратко опишите его назначение и ссылку на регламент. Так любой участник понимает, что искать внутри `docs/` и где лежит источник правды по архитектуре и процессам.

## Связь с агентами

- **50-docs-update** читает реализацию и результаты (`local/**`, `results/**`, `inputs/**`) и обновляет документы в `docs/**`.
- Запускайте этого агента после успешных шагов реализации и качества (`30-code-build`, `40-tests-run`, `40-code-review`), но до поставочных агентов `60-release-build` и `60-bitrix-update-build`.

Подробности об остальных агентах и последовательности работы см. в корневом `README.md`.
