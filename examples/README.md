# Каталог examples

Эта директория хранит все демонстрационные материалы проекта, чтобы агенты могли быстро воспроизводить сценарии, ссылаться на эталонные проекты или прикладывать вспомогательные файлы. Перед выполнением батча стоит проверить, нет ли уже нужного артефакта внутри `examples/`.

## Структура

- `snapshots/` — снимки данных в формате `.jsonl` (HL-блоки, модули, настройки). Их можно импортировать в тестовое окружение или использовать как фикстуры в автоматических тестах. Каждому файлу требуется краткое описание в `docs/` или `inputs/`. исопльзуетя файл scan.php для получения снепшота.
- `links/` — полноразмерные проекты и связанные ресурсы (например, `simai.property`). Используйте их как референс для UI или интеграций; при добавлении указывайте источник и версию.
- `files/` — небольшие вложения: SQL-дампы, CSV, листинги, которые не подходят под снапшоты, но нужны для требований или cookbook.

## Правила пополнения

1. Перед добавлением нового артефакта убедитесь, что он не нарушает лицензии и не содержит чувствительных данных.
2. Документируйте назначение артефакта в соответствующем требовании (`inputs/**`) или инструкциях (`docs/**`).
3. Если структура каталога расширяется, обновите корневой `README.md` и этот файл.
